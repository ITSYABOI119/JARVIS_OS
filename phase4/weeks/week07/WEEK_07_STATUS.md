### The bug adversarial review caught (would have been a dark monitor)

The FB was first written to map **Write-Combining** via `vspace_map_pages(..., seL4_X86_WriteCombining)`. But `vspace_map_pages`' last arg is `int cacheable` (libsel4utils maps `1 ‚Üí WriteBack`, `0 ‚Üí Uncached`), **not** a `seL4_X86_VMAttributes` ‚Äî so passing the WC enum value (4) is *truthy* ‚Üí a **WriteBack/cacheable** mapping, which the GPU scanout never sees (no cache snoop) ‚Üí blank screen. Fixed by passing **`0` (Uncacheable)** ‚Äî MTRR-independent and guaranteed visible, the same device-memory mapping the NVMe BAR0/DMA path uses. True WriteCombining (faster) needs a raw `seL4_X86_Page_Map` path ‚Äî deferred to 2c.
