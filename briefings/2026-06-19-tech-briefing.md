# JARVIS Daily Tech Briefing — Friday, June 19, 2026

**Lead — A verification-aware DSL that compiles to *readable* verified C (LCTES'26, presented this week).** "Towards Verifiable System Code using a DSL Compiled to Efficient and Readable C Code" describes a DSL for critical embedded software that carries source-level proofs (proof-by-deduction) and emits C that's both efficient and human-readable — with a published Zenodo artifact. *Why it matters:* JARVIS deliberately ships a *functional-but-unverified* seL4 x86-64 config (`KernelFastpath=ON`, outside the verified X64 proof) and hand-writes a lot of bare-metal C — drivers, shmem IPC, the inference/qdot kernels. A toolchain that yields verified-yet-maintainable C is exactly the angle that could, longer term, claw back assurance on those hand-written paths without forcing proofs against opaque generated code. Research-stage and embedded-focused — a reference to read, not adopt. [LCTES'26 (ACM DL)](https://doi.org/10.1145/3814943.3816170) · [artifact](https://doi.org/10.5281/zenodo.20086279)
