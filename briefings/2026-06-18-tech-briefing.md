# JARVIS Daily Tech Briefing — Thursday, June 18, 2026

**Lead — Linux is set to make the x86 TSC a hard requirement.** Now that i486/K5/Elan support is being dropped, a patch queued in the tip tree for the Linux 7.2 merge window strips the optional non-TSC code paths and assumes the Time Stamp Counter is always present on x86. *Why it matters:* JARVIS's bare-metal x86-64 build already leans on RDTSC for Stage-5 timing and assumes TSC exists — mainline formally codifying "TSC always on x86" validates that assumption, so the non-TSC fallback you'd otherwise carry is provably unnecessary on any CPU JARVIS targets (Ryzen 2700X/5600 both have invariant TSC). [Phoronix](https://www.phoronix.com/news/Linux-Kernel-TSC-Unconditional)

## Systems / OS

- **Linux 7.2 merge window is open (first RC ~Jun 28, stable ~mid/late August).** Headline items: unconditional x86 TSC (above), retiring AMD K5/Elan + orphaning Geode, initial Apple M3 boot, cache-aware scheduling, and Switchtec PCIe Gen6 switch support. Becomes the default kernel for Ubuntu 26.10 / Fedora 45. [Phoronix](https://www.phoronix.com/news/Linux-7.2-Features-Expected)
- **Cache-aware scheduling lands in 7.2.** The scheduler now factors LLC locality into task placement. *Why it matters:* directly adjacent to your M2/M3 SMP work — you hand-place PA/PB and (soon) the M3 worker pool with explicit `seL4_TCB_SetAffinity` because seL4 won't auto-distribute, so Linux's approach to keeping hot threads on cache-warm cores is a useful reference for how you pin the threadpool. [Phoronix](https://www.phoronix.com/news/Linux-7.2-Features-Expected)
