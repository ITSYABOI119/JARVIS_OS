## bench_engine.c Design

### CLI

```
bench_engine <model.gguf> [--tokens N]
```

- `model.gguf`: path to GGUF model file (required)
- `--tokens N`: generation tokens per prompt (default 64)

### Load Phase

1. `fopen` + `fread` entire GGUF into memory (same pattern as `test_gemma4_native.c`)
2. `gguf_open_memory()` to parse GGUF header
3. `qmodel_load()` to set up zero-copy quantized tensor refs
4. `gguf_vocab_extract()` + `gguf_vocab_init_tokenizer()` for tokenizer
5. `llama_alloc_state()` for activation buffers + KV cache
6. Time with `clock_gettime(CLOCK_MONOTONIC)`, report in seconds
7. Read `/proc/self/status` VmHWM for peak RSS (Linux only, `#ifdef __linux__`)

### Prompt Construction

**10 fixed prompts** (same as `bench_models.sh` quality bench):

```c
static const char *prompts[] = {
    "The seL4 microkernel is",
    "Explain how virtual memory works in three sentences.",
    "What is the difference between a process and a thread?",
    "Write a C function that reverses a string in place.",
    "The capital of Australia is",
    "What are the key benefits of formally verified software?",
    "Describe the TCP three-way handshake step by step.",
    "What happens when you type a URL into a browser?",
    "Compare RISC and CISC architectures in two sentences.",
    "Write a bash one-liner to find all .c files larger than 1MB.",
};
```

**Architecture detection** (from `qm.config`):

- If `config.embed_scale` is true (Gemma): wrap with chat template
  - Exact token IDs from `inference_server.c:65-98`:
    `[bos, 105, 2364, 107, ...encoded_text..., 106, 107, 105, 4368, 107, 98]`
  - Stop on `eos_token = 1` (NOT `tok.eos_id` which is 106 for Gemma)
- Else (Llama/other): prepend `tok.bos_id`, stop on `tok.eos_id`

### Timing: Manual Prefill + Generation Loop

`qmodel_generate()` combines prefill and generation in one call with no way to split timing. For separate measurements, use the manual pattern from `bench_inference.c`:

```c
/* KV reset */
state.pos = 0;
size_t kv_bytes = (size_t)qm.config.n_layers * state.max_seq_len *
                  qm.config.n_kv_heads * qm.config.head_dim * sizeof(float);
memset(state.key_cache, 0, kv_bytes);
memset(state.value_cache, 0, kv_bytes);

/* Phase 1: Prefill (timed separately) */
double t0 = time_ms();
for (int i = 0; i < n_prompt; i++)
    qmodel_forward(&qm, &state, prompt_ids[i]);
double prefill_ms = time_ms() - t0;

/* Phase 2: Generation (timed separately) */
t0 = time_ms();
for (int g = 0; g < max_gen; g++) {
    int next = sample_greedy(state.logits, qm.config.vocab_size);
    out_ids[g] = next;
    if (next == eos_token) { n_gen = g + 1; break; }
    qmodel_forward(&qm, &state, next);
    n_gen = g + 1;
}
double gen_ms = time_ms() - t0;
```

### Error Handling

- If model fails to load: print GGUF `general.architecture` string if parseable, print error, `exit(1)`
- If tokenizer fails: print error, `exit(1)`
- If state alloc fails (OOM): print error with expected memory, `exit(1)`
- Do NOT crash on any failure — the wrapper script continues to next model

## bench_engine_models.sh Design

```bash
#!/bin/bash
# JARVIS AI-OS — Engine Bench (all models)
# Usage: bash phase3/scripts/bench_engine_models.sh
```

1. **Header**: date, hostname, CPU (`/proc/cpuinfo`), RAM (`/proc/meminfo`)
2. **Auto-compile**: if `/tmp/bench_engine` doesn't exist or source is newer, compile with:
   ```bash
   gcc -O2 -mavx2 -mfma -std=c11 -D_POSIX_C_SOURCE=200809L -D_GNU_SOURCE \
       -I phase3/src/ai \
       phase3/src/ai/bench_engine.c phase3/src/ai/llama_quant.c \
       phase3/src/ai/llama_load.c phase3/src/ai/llama_forward.c \
       phase3/src/ai/gguf_parser.c phase3/src/ai/gguf_vocab.c \
       phase3/src/ai/tokenizer.c phase3/src/ai/tensor_ops.c \
       phase3/src/ai/dequant.c phase3/src/ai/sampling.c \
       phase3/src/ai/inference.c \
       -lm -o /tmp/bench_engine
   ```
3. **Model loop**: `find models/ -maxdepth 1 -name "*.gguf" -type f | sort`
4. **Per model**: `set +e`, run `/tmp/bench_engine "$model"`, capture stdout
5. **Separator**: blank line + dashes between models
6. **Output**: tee to `models/jarvis_engine_bench.txt`
